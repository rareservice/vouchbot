üåç General Commands

+vouch Alias: +rep

Records a transaction vouch. This puts the vouch in "Pending" status until verified.

Primary Usage +vouch @User legit got <item> for <price> TYSMM!
Secondary Usage +rep @User legit got <item> for <price> TYSMM!
Example +rep @Seller legit got nitro for $2 tysmm!
  • Bot auto-DMs the seller requesting proof.
  • Alias +rep works exactly the same as +vouch.
+show_profile Alias: +p, +profile

Displays a comprehensive user profile card including reputation, badges, and shop info.

Usage +show_profile [User]
Short Usage +p [User]
  • Displays Trusted/King/Scammer badges.
  • Shows the last 5 verified vouch comments.
  • Displays "Frozen" status if user is a suspect.
/setshop Slash Only

Links your Discord shop server to your profile card.

Usage /setshop <Invite Link>
/setproducts Slash Only

Updates the "Products" field on your profile card to list what you sell.

Usage /setproducts <List of items>
+reset

Cancels all your currently pending vouches. Useful if a buyer made a mistake.

Usage +reset
+advance_lookup Alias: +advance

Checks if a user has an active Premium subscription.

Usage +advance_lookup @User
Short Usage +advance @User

üíé Premium Commands & Safety

+report Premium

Opens a "Suspect Report" ticket against a user. Requires attaching proof.

Usage +report <Accused ID> <Victim ID> <Reason>
  • Creates a private ticket for Staff.
  • If Valid: User gets "Suspect" badge.
+scammer Alias: +scam

Opens a high-priority "Scammer Report" ticket. Use for confirmed scams. You must attach proof when using this command.

Usage +scammer <Accused ID> <Victim ID> <Explanation> (+Attach Proof)
Secondary Usage +scam <Accused ID> <Victim ID> <Explanation> (+Attach Proof)
  • If Valid: User gets "Scammer" badge & Role.
  • User is "Frozen" and cannot receive vouches.
/appeal Premium

Opens an appeal form if you have been marked as a Scammer or Suspect.

Usage /appeal
  • Admins can Accept (Clean) or Deny the appeal.
